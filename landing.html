How it works

From click to Issue, in four steps.

Here's the path your user's feedback takes from the moment they click the button to the moment it lands in your backlog.

1

User clicks "See a Bug?"

Drop the floating button anywhere in your app. It opens the modal without interrupting the user's flow.

<!-- anywhere in your app --> <button onclick="openFeedback()"> See a bug? </button>
2

Collect what matters

A clean modal gathers a title, description, type (bug, idea, feedback), and optional email — plus browser metadata you didn't have to ask for.

{ title: "Save button broken on /settings", type: "bug", email: "user@acme.com", meta: "Chrome 124 · macOS · 1440×900" }
3

POST to your backend

The component sends a JSON payload to /feedback/:owner/:repo. A tiny Hono server validates it and forwards to GitHub.

POST /feedback/acme/web-app content-type: application/json { "title": "...", "body": "...", "labels": ["feedback"] }
4

A new Issue appears

Your backend uses its GITHUB_API_KEY to create the Issue in the target repo — labeled, formatted, and ready to triage.

✓ Issue #142 created in acme/web-app ✓ Labeled: feedback, bug ✓ Linked in user email (optional)   github.com/acme/web-app/issues/142
Get started

Three steps. One config object.

Install, set your target repo, and embed the component. No build step, no bundler config.

  • Works with any frontend stack — React, Vue, Svelte, or plain HTML
  • One backend endpoint can serve every project you own
  • Deploy to Vercel, Cloudflare, or run locally with Bun
  • MIT-licensed — fork it, extend it, ship it
config.js
terminal
.env
// generic-component.html — drop in and configure
const CONFIG = {
  owner:    'your-github-username',
  repo:     'your-repo-name',
  endpoint: '/feedback',
};

// that's it. The modal will POST to:
// /feedback/your-github-username/your-repo-name
# 1. clone & install
$ git clone https://github.com/you/feedback-pusher
$ cd feedback-pusher
$ bun install

# 2. run the server
$ bun github/generic-index.js
→ listening on :3000
# .env
GITHUB_API_KEY=ghp_xxxxxxxxxxxxxxxxxxxx

# Needs `repo` scope on your PAT
# The key stays server-side — never shipped to the browser
